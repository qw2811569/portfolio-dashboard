## Design（5 題）

| 題                                   | 決議                                                                                       | 理由                                                                               |
| ------------------------------------ | ------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------- |
| Q-D1 · Contract / Zod parse mode     | **strict reject / fail-closed**                                                            | Contract boundary = trust boundary；coerce 會把壞資料默默帶進 store / restore lane |
| Q-D2 · Freshness / StaleBadge 閾值   | **targets=7d, fundamentals=30d, macro/calendar=1d, restore artifact=show exact timestamp** | 與既有 freshness 決議相容，degraded mode 最清楚                                    |
| Q-D3 · Signed URL TTL                | **15m**                                                                                    | 支援 dashboard 讀取與 retry，不會把 private artifact 暴露太久                      |
| Q-D4 · Accuracy Gate failure UX      | **hard-block + explicit reason**                                                           | fail-closed 才符合 internal beta truthfulness；placeholder 易誤當資料慢            |
| Q-D5 · Four-persona / scoring 可見度 | **分數 + 簡短理由；不露完整 trace**                                                        | 完整 trace 讓 UI 失焦，也增加提示/合規風險                                         |
