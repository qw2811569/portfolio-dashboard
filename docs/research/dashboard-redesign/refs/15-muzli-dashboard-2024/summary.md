### 3. 重複出現的 studio / pattern hint

- **RonDesignLab** 占 11/39 — 跟 Refs 01/07/08/09/10 是同一支線；表示這支線設計在 muz.li 編輯心目中是「dashboard 最強標竿」。
- **One Week Wonders (OWWStudio)** 出現 4 次 — 另一個值得追的 studio。
- **Outcrowd** 出現 2 次 — 金融 SaaS 主力。
- **Purrweb** 出現 2 次 — mobile 金融專業。
- **Geex Arts** 出現 3 次（同一系列三件） — 一致 brand 學習對象。

→ **建議下一波 ref**：直接挖 `Outcrowd / OWWStudio / Geex Arts` 在 Dribbble 的金融類作品 portfolio。

### 4. 跟 muz.li 作者觀點對齊本專案的態度

作者的核心主張：「**第二階段刻意去看 quirky / 不實用的設計，再回頭篩**」。

對應本專案：

- 我們現在的痛點是「資訊太多難聚焦」 — 看「不實用但有極端視覺紀律」的 ref（Ref 01 Ron Design IG · 本文的 12 Sphere UI · 本文的 26 Edtech 等）反而能撞出**降階思路**。
- 不是每個 entry 都該抄 — 但每個都該問「這份犧牲了什麼資訊密度，換了什麼聚焦力？」

### 5. 待 Round 1 給 Codex 的取捨題

1. **muz.li 39 entry → 9 個 route page baseline**：要不要把「對本專案最相關的 7 個」的 hero image 並排，叫 Codex 挑 3-4 種「明顯不適合 / 明顯該抄」的，理由 grep 既有 component / token 給 file:line。
2. **Studio-deep-dive**：Outcrowd / OWWStudio / Geex Arts 三家 portfolio 要不要再挖 5-10 件作為 Refs 16-30 候補？
3. **2026 升級版**：muz.li 同主題的 [2026 文章](https://muz.li/blog/best-dashboard-design-examples-inspirations-for-2026/) 是否值得也爬，比對 1 年內 dashboard trend 的位移？
