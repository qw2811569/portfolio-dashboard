# Claude 台股分析工具組合與使用手冊

最後更新：2026-03-24

## 1. 先講結論

如果目標是讓 Claude 更會判斷台股，不要只找「prompt skill」。
真正有價值的是：

- 台股資料源
- Claude 可調用的 MCP server
- 本地 fallback library

嚴格來說，下面列的多數是 MCP 或資料 repo，不是官方 `skill`。
但對這個專案來說，它們比純文字 skill 更實用。

## 2. 推薦組合

### 2.1 最推薦的核心組合

#### A. `twsemcp`

Repo:

- https://github.com/pyang2045/twsemcp

定位：

- 最適合先接進 Claude 的台股 MCP
- 直接連台灣證交所 OpenAPI
- 適合收盤分析、盤後檢查、快速查公司財務與市場資料

優點：

- 官方資料源，可信度高
- 可直接給 Claude Desktop / MCP client 用
- 支援股價、成交量、財報、月營收、法人、ETF、產業統計、交易日曆

最適合的場景：

- 每日收盤分析
- 盤後異常波動說明
- 事件追蹤的事實查核
- 快速確認某檔個股基本數據是否和模型推論一致

注意：

- 以 TWSE 官方資料為主
- 若需要大量上櫃 / 更細緻歷史欄位，還要搭配其他資料源

#### B. `FinMind`

Repo:

- https://github.com/FinMind/FinMind

定位：

- 最適合當台股深度研究的資料底座
- 不是現成 Claude skill，但非常適合拿來做你自己的分析工具層

優點：

- 台股為主，資料面廣
- 有股價、月營收、財報、法人、融資融券、期貨、選擇權、新聞等資料
- 適合做歷史回溯、題材驗證、研究報告補強

最適合的場景：

- 深度研究
- 同業比較
- 催化劑與風險驗證
- 回頭檢查 AI 的論點是否有數據支撐

注意：

- 不是裝上去就能直接讓 Claude 用
- 最適合接成你自己的 research adapter 或 MCP

#### C. `twstock`

Repo:

- https://github.com/mlouielu/twstock

定位：

- 輕量、本地可跑的台股 Python library
- 適合做 fallback 與簡單技術面計算

優點：

- 支援 TWSE 與 TPEX
- 本地整合容易
- 適合補即時報價、歷史資料、均線、基本技術面邏輯

最適合的場景：

- 本地 fallback
- 快速驗證個股價格 / 均線 / 技術面
- 在沒有完整 MCP 時先做最小可用版

注意：

- 比較像 library，不是現成 Claude 工具
- 有 request limit，要控制查詢頻率

### 2.2 建議的補強組合

#### D. `mcp-tradingview-server`

Repo:

- https://github.com/bidouilles/mcp-tradingview-server

定位：

- 技術面補強工具
- 適合讓 Claude 補看 OHLCV、型態、技術指標

最適合的場景：

- 深度研究中的技術面段落
- 判斷支撐 / 壓力 / 趨勢是否轉折
- 驗證「現在適不適合追價 / 等拉回」

注意：

- 台股 symbol mapping 要實測
- 它比較適合補技術面，不應該取代台股本地資料層

#### E. `mcp-stockscreen`

Repo:

- https://github.com/twolven/mcp-stockscreen

定位：

- 條件選股與觀察清單工具
- 適合幫 Claude 做選股掃描與比較名單

最適合的場景：

- 找相同題材候選股
- 找強勢股 / 弱勢股
- 研究時建立比較池

注意：

- 更適合做 screening，不是台股專用
- 若用在台股，ticker 規則要先驗證

## 3. 我建議你的實際安裝順序

### 方案 A：最實用、最低成本

- 先接 `twsemcp`

適合：

- 你想最快提升 Claude 在台股上的判斷力
- 你最在意的是收盤分析、事件追蹤、盤後事實核對

### 方案 B：最適合你現在這個專案

- `twsemcp` + `FinMind` + `twstock`

理由：

- `twsemcp` 負責官方即時 / 盤後資料與快速查核
- `FinMind` 負責研究深度與歷史驗證
- `twstock` 負責本地 fallback 與簡單技術計算

這一組最適合你的 `收盤分析 + 深度研究 + 本地部署` 架構。

### 方案 C：進階版

- `twsemcp` + `FinMind` + `twstock` + `mcp-tradingview-server` + `mcp-stockscreen`

適合：

- 你想讓 Claude 同時能做
- 台股事實查核
- 技術面判讀
- 條件選股
- 研究比較池

缺點：

- 接線較多
- 需要更嚴格的資料優先序，避免來源打架

## 4. Claude 使用規則

下面這段是給 Claude 在分析時遵守的工作方式。

### 4.1 資料優先序

Claude 在引用資料時，優先序應固定為：

1. `twsemcp` / TWSE 官方資料
2. `FinMind`
3. `twstock`
4. `TradingView`
5. `Yahoo Finance / stockscreen`
6. 模型推論

規則：

- 若不同來源數字不一致，優先採用較新的官方資料
- 若不是直接數據，而是推論，必須明寫「這是推論」
- 不能把技術面判斷寫成基本面事實

### 4.2 產出格式規則

Claude 產出時要分成三層：

1. 事實
2. 解讀
3. 動作建議

範例：

- 事實：今日收盤價、成交量、月營收、外資買賣超
- 解讀：量價是否背離、題材是否延續、財報預期是否改善
- 動作建議：續抱 / 減碼 / 等拉回 / 停損觀察點

## 5. 收盤分析怎麼用

### 5.1 目標

收盤分析的重點不是重做一份深度報告，而是：

- 今天哪裡和預期一致
- 哪些持股異常
- 哪些事件該重新評估
- 明天要先盯什麼

### 5.2 建議資料流

收盤分析時，Claude 應優先使用：

1. App 內目前持倉、事件、daily report history、brain
2. `twsemcp` 查今天收盤、成交量、估值指標、法人 / 市場資訊
3. 必要時才用 `twstock` 補價格 / 技術面
4. 只有出現明顯異常時，才補 `TradingView`

### 5.3 收盤分析應回答的問題

- 今天總損益變動最大的 3 檔是什麼
- 漲跌是否符合原本策略與事件預期
- 是否出現異常量價 / 籌碼 / 事件反應
- 哪些部位需要列入 `needsReview`
- 哪些事件要從 `pending` 改為 `tracking` 或準備 `closed`

### 5.4 收盤分析的輸出模板

建議 Claude 每次固定輸出：

#### A. 今日總結

- 組合整體表現一句話

#### B. 異常持股

- 只列最重要的 1-3 檔
- 每檔都要有「事實 + 解讀 + 動作」

#### C. 事件追蹤

- 哪些事件與股價反應一致 / 不一致

#### D. 明日優先觀察

- 最多 3 點

### 5.5 收盤分析不該做的事

- 不要把所有持股都重寫成長報告
- 不要為了完整而塞太多產業背景
- 不要在沒有新資料時重複昨天的結論

## 6. 深度研究怎麼用

### 6.1 目標

深度研究的重點是：

- 把一檔股票或整個組合研究深
- 找出催化劑、風險、時程、估值與操作節點
- 最後能回寫到策略大腦與事件系統

### 6.2 建議資料流

深度研究時，Claude 應使用：

1. App 內目前持倉、事件、brain、analysisHistory、researchHistory
2. `FinMind` 補歷史營收、財報、法人、融資融券、期貨 / 選擇權資料
3. `twsemcp` 做官方數據交叉驗證
4. `TradingView` 補技術面結構
5. `mcp-stockscreen` 找可比較同業 / 題材股

### 6.3 單股深度研究應回答的問題

- 公司 / 題材的核心邏輯是否還成立
- 未來 1-3 個月催化劑是什麼，時間點在哪
- 最可能的 3 個風險是什麼
- 估值偏高還偏低，和誰比
- 目前這個持倉該續抱、加碼、減碼還是退出

### 6.4 全組合研究應回答的問題

- 哪些部位最需要立即動作
- 資金配置是否失衡
- 哪些策略規則已經過時
- 哪些錯誤在反覆發生
- 下週最該執行的 5 個動作是什麼

### 6.5 深度研究的輸出模板

建議 Claude 固定輸出：

#### A. 核心結論

- 先講結論與信心度

#### B. 多頭邏輯

- 2-4 點

#### C. 風險與失敗條件

- 2-4 點

#### D. 時間軸

- 未來 1 週 / 1 個月 / 1 季

#### E. 操作計畫

- 進場區
- 加減碼條件
- 停損 / 失敗條件
- 追蹤指標

#### F. 是否回寫策略大腦

- 新規則
- 新教訓
- 要刪的舊規則

## 7. 給 Claude 的簡版操作守則

如果 Claude 真的要直接照著做，建議遵守下面這段：

1. 收盤分析優先求「短、準、可行動」，不要過度研究。
2. 深度研究優先求「邏輯、數據、時間點」，不要只寫情緒化看法。
3. 沒有資料就明說缺資料，不要假裝知道。
4. 若不同來源衝突，先標示差異，再採用官方資料。
5. 每次建議都要附上條件，而不是只說看多或看空。
6. 若研究結果足以改變策略，必須提出可回寫到 brain 的規則或教訓。

## 8. 建議下一步

如果要真正落地，下一步建議是：

1. 先試裝 `twsemcp`
2. 若需要研究深度，再把 `FinMind` 接成 research adapter
3. 再決定要不要補 `TradingView` 與 `stockscreen`

對這個專案來說，最值得先做的是：

- 讓 `deep research` 先能吃 `FinMind`
- 讓 `daily analysis` 先能吃 `twsemcp`
- 保留 `twstock` 當本地 fallback
