# AnythingLLM 台股研究提問模板

## 使用原則

1. 先要求它區分：已知事實、推論、待驗證事項。
2. 先問整理與比較，再問結論。
3. 如果牽涉時間敏感資訊，要明講資料新鮮度不足時不能硬下結論。
4. 技術面、籌碼面、產業面、事件面要分段回答，避免混成一句大結論。

## 模板 1：策略總覽整理

請根據目前知識庫中與台股策略有關的文件，整理一份高密度摘要，分成：

- 產業趨勢
- 事件節奏（法說、財報、月營收）
- 消息與股價連動
- 籌碼與技術分析
- 目前還缺什麼資料

要求：

- 先列已知事實
- 再列推論
- 最後列待驗證事項
- 不要把缺資料的地方說成結論

## 模板 2：產業鏈與消息連動

請用台股語境分析某個產業或題材，回答：

- 目前主線是什麼
- 受惠鏈條有哪些
- 哪些公司可能是領頭股、跟漲股、補漲股
- 下一個重要驗證點是什麼
- 什麼情況下這條邏輯會失效

如果知識庫沒有足夠證據，請直接說明不足處。

## 模板 3：籌碼與技術面

請根據知識庫內容，整理台股常見的籌碼與技術分析框架：

- 量價結構
- 均線與位置
- 法人 / 主力 / 融資融券
- 技術面如何和事件面一起解讀
- 哪些情況最容易產生假突破或誤判

## 模板 4：單一標的策略檢視

請針對某一檔台股標的，從下列角度做研究框架：

- 產業定位
- 近期可能催化劑
- 需要追蹤的數據
- 技術面觀察重點
- 籌碼面觀察重點
- 可能的錯誤劇本

回答時請區分：

- 知識庫已有的證據
- 尚未建立的證據

## 模板 5：把雜資料變成研究提綱

請根據知識庫裡與指定主題相關的資料，整理成一份研究提綱：

- 主題定義
- 關鍵變數
- 需要補的資料
- 可觀察的市場反應
- 可驗證的判斷條件

## 模板 6：客戶報告前置整理

請從知識庫中整理出可用於客戶報告的內容，但不要直接產出華麗文案。先整理：

- 目前投組主線
- 產業與持股的關聯
- 風險集中在哪
- 哪些是有資料支撐的結論
- 哪些需要人工補判讀

## 模板 7：策略反證

請針對某個策略主張，列出：

- 支持它的證據
- 反對它的證據
- 哪些是還沒被驗證的假設
- 最容易導致誤判的台股特性

## 模板 8：下一步研究待辦

請根據知識庫目前內容，給我一份「最值得補充的 5 個研究空缺」：

- 每項空缺缺什麼
- 為什麼重要
- 補齊後能改善哪一類判斷
