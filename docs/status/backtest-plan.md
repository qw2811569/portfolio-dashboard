| 6    | 2025-03-01 | 新年度 Q1 初段，重新評價與春季資金切換         |

| 10   | 2026-03-01 | 最新近端環境，驗證規則是否還適用於當前市場     |

### 這 10 回想覆蓋的市場環境

這組日期的目標不是追求每次都剛好遇到大事件，而是要平均覆蓋這幾種情境：

1. **年初重新定價期**
2. **AI / 電子題材升溫期**
3. **夏季或 Q3 震盪期**
4. **財報 / 法說 / 作帳前後切換期**
5. **年底展望切換期**
6. **最新近端市場環境**

如果後面要再強化事件密度，可以額外插入「特定事件日」版本，例如：

- 月營收公布密集週
- 聯準會 / 央行決議週
- 主要法說會前後
- 大盤明顯修正日

## 五、執行提醒

### 1. 先解決 metadata 分類不一致

目前你口頭指定的幾檔，和 `STOCK_META` 內建分類有幾個不一致：

- **3017 奇鋐**：目前 metadata 是成長股，不是景氣循環
- **3231 緯創**：目前 metadata 是成長股，不是景氣循環
- **00637L**：目前 metadata 是 ETF/指數，不是權證

建議在正式開始回測前，先決定：

- 要以 **人工指定分類** 為準
- 還是以 **repo metadata** 為準

我的建議是：**以 repo metadata 為準**，避免之後回測程式和人工解讀打架。

### 2. 回測輸出至少要記錄這些欄位

每次回測應記：

- 日期
- 股票代碼
- 帳戶
- 策略類型
- analysis framework
- 注入的 knowledge ids
- 主要判斷
- 預測方向
- confidence
- 實際結果
- 規則命中 / 失誤

### 3. 後續可直接接 confidence evolution

這份 backtest plan 的目的，不只是做報告，而是要讓後面可以直接做：

- 規則命中率統計
- confidence 調整
- 不同策略類型的規則排名
- cold-start baseline vs dynamic injection 效果比較
