台股投資工作台 · 專案框架說明

把決策過程變成可追蹤、可累積的資產

一個整合持倉、事件、AI 分析、策略記憶的閉環投資作業系統

核心定位：這不是看盤工具，也不是記帳 App。它是一個讓投資者的每一次決策都有脈絡、可追蹤、可回顧的工作台——讓「部位、催化事件、研究、復盤、策略記憶」形成完整閉環。
為什麼需要這個

主動投資者面臨的真實困境

這些問題在使用 LINE 群組、Excel、截圖筆記本的投資者身上每天都在發生。

📊

部位管理分散，無法即時掌握

多組合、多帳戶的持倉散落在券商 App、截圖、Excel 各處，很難在需要決策的當下有完整視圖。

統一部位管理，支援截圖自動解析回寫
📅

決策事後說不清楚「當初為什麼買」

投資邏輯只存在腦袋裡，沒有結構化記錄。復盤時無從比較「預期 vs. 實際」，無法真正學習。

催化事件追蹤，讓每筆持倉都有論點記錄
🤖

每日收盤分析耗時，品質不穩定

要同時看財報、技術面、籌碼、新聞，人工整合費時又容易遺漏，AI 工具又沒有你的持倉 context。

AI 自動整合持倉 context，每日生成結構化復盤
🧠

教訓反覆犯，成功模式無法複製

每次止損後說「下次不這樣」，但沒有機制讓這句話變成具體的規則，下次還是會重蹈覆轍。

策略大腦沉澱成功 / 失敗案例，讓 AI 下次直接引用
🔍

研究成果孤立，無法反哺決策

花時間做的個股研究存在文件夾裡，下次要用時找不到，或者已經和當前持倉 context 脫節。

研究與持倉整合，歷史脈絡自動帶入下一次分析
📋

客戶溝通缺乏可信度與透明度

對客戶說「看好這檔」，卻沒有具體的催化劑、進場邏輯、停損條件。說服力和可信度都不足。

可輸出結構化決策報告，包含論點、事件、追蹤結果
核心功能

六個模組，形成一條完整的決策鏈

每個模組獨立可用，但設計目的是讓它們協同運作、互相餵養。

💼

智慧持倉管理

多組合、多帳戶的持倉統一管理。支援截圖 OCR 解析自動回寫部位，即時計算損益與成本結構。

持倉 / 損益 / 多組合
⚡

催化事件追蹤

將投資論點與具體的「催化事件」掛鉤。事件從待定 → 追蹤 → 結束自動流轉，每一筆持倉的進場邏輯都有可查記錄。

事件生命週期 / 法說 / 月營收 / 政策
📈

每日 AI 收盤分析

AI 自動整合持倉現況、待追蹤事件、歷史分析脈絡，生成結構化的收盤復盤報告，對比預期與實際，輸出明天的觀察重點。

AI 復盤 / 結構化報告 / 盲測評分
🧠

策略大腦（知識沉澱）

將每次的成功邏輯與失敗教訓沉澱成可供 AI 調用的結構化規則。下次 AI 分析時，會自動引用相關的歷史案例與驗證過的策略。

規則引擎 / 歷史案例 / 策略記憶
🔬

深度研究整合

在同一平台對特定標的發起深度研究，系統自動帶入該標的的歷史分析、持倉記錄與相關事件，讓每次研究都站在之前工作的肩膀上。

個股研究 / 脈絡整合 / 供應鏈
📄

客戶報告輸出

將投資決策從「個人看法」轉化為包含持倉論點、催化事件時間軸、風險因素與追蹤結果的結構化報告，提升對客戶的透明度。

週報 / 決策透明 / 可追蹤
運作邏輯

一條從「建倉」到「知識沉澱」的閉環

每一個步驟的產出，都是下一個步驟的輸入。系統的價值隨使用時間累積，不是用了就歸零。

1

建立持倉 + 投資論點

上傳交易截圖或手動輸入，建立真實部位。同時記錄「為什麼買這檔」——催化事件、目標價格、停損條件。

截圖 OCR 解析 多組合支援 催化事件登記
2

市場資料自動同步

系統自動同步 TWSE 收盤價格、法人籌碼、公開資訊（MOPS 重大訊息）與財報數據，確保每次分析都基於最新資料。

TWSE 收盤報價 法人買賣超 MOPS 公告 月營收
3

事件生命週期追蹤

系統根據日期與價格變動，自動追蹤事件進展。法說前 / 法說後、月營收公布前後，事件狀態自動流轉並記錄結果。

待定 → 追蹤 → 結束 法說 / 財報 / 政策 事件窗口價格記錄
4

每日 AI 收盤分析

AI 整合當前持倉、未結事件、歷史脈絡與市場知識庫，生成結構化復盤報告。包含「預期 vs 實際」比對與明日觀察重點。

結構化復盤 策略知識庫引用 明日行動清單
5

知識沉澱入策略大腦

每次分析結論、事件教訓、成功 / 失敗案例，都以結構化格式回寫至「策略大腦」。下一次 AI 分析時，自動帶入相關歷史規則。

成功 / 失敗案例庫 策略規則引擎 知識隨時間累積
6

深度研究 + 輸出報告

基於累積的持倉脈絡與歷史分析，對個股發起深度研究，並輸出可對客戶呈現的結構化決策報告，形成完整閉環。

個股深度研究 週報 / 客戶報告 決策閉環完成
UX 實際畫面

使用者每天實際怎麼用這個 App

以下為實際系統截圖（2026/03/30），可直接用來向老闆展示使用者的主流程與頁面體驗。

步驟 1 · 持倉頁

先看部位全貌與風險結構

使用者每天先進持倉頁，快速掌握總成本 / 總市值 / 持股數、產業集中度、強弱股與持股明細，確認今天該盯哪些部位。

步驟 2 · 事件頁

把投資論點轉成可追蹤的事件

進入行事曆頁後，使用者會先看接力計畫與事件標籤，再逐筆檢視法說 / 財報 / 催化事件，確認哪些風險與催化已進入決策窗口。

步驟 3 · 收盤分析報告

收盤後看 AI 復盤與明日行動依據

在收盤分析頁，使用者查看當日報告摘要、持倉漲跌與異常波動，讓「今天發生什麼、接下來怎麼做」有結構化依據可留存與回顧。

可靠度

為什麼可以信任這套系統

從資料完整性、錯誤隔離到 AI 輸出品質，都有對應的機制保護。

🛡 資料安全與持久性

  • 持倉資料雙重存儲：localStorage + Vercel Blob 雲端
  • Debounce 自動儲存，操作不會無聲消失
  • 支援本機備份匯出 / 匯入，不依賴單一存儲
  • Schema 升級時有 migration patch 機制

🔒 UI 錯誤隔離

  • Panel 級 ErrorBoundary，單一區塊崩潰不影響其他面板
  • Header 區塊有獨立錯誤邊界
  • AI 分析失敗不會影響持倉顯示
  • 雲端同步失敗降回本地模式，不中斷操作

🧪 程式碼品質保障

  • Vitest 單元測試覆蓋核心邏輯
  • Playwright E2E 測試覆蓋主要工作流程
  • ESLint + Prettier 強制代碼規範
  • 本地 verify 指令，每次交付前強制過關

🤖 AI 輸出品質控管

  • 分析 prompt 帶入完整持倉 context，非憑空生成
  • 策略知識庫只注入驗證過的歷史案例
  • 結構化輸出格式，禁止模糊描述
  • 盲測評分機制，比對預測 vs 實際結果

📡 資料新鮮度管理

  • TWSE 報價 TTL 3-5 分鐘快取
  • 法人籌碼 TTL 15 分鐘
  • 過期財報 / 舊分析不會當成當下結論
  • API 失敗指數退避重試，不直接崩潰

🔄 多 AI 協作可靠性

  • Claude / Qwen / Gemini 分工明確，不互相覆蓋
  • 每次交付前有 canonical 交接格式
  • AI 狀態儀表板即時顯示各 AI 工作狀態
  • 唯一 canonical 規則文件，無版本分歧

總結

三件事說清楚這個專案

🎯

是什麼

台股投資決策工作台。把部位、事件、AI 分析、策略記憶串成閉環，讓每一次投資決策都可追蹤、可復盤、可累積。

⚙️

怎麼運作

六個模組協同運作：持倉管理 → 事件追蹤 → 資料同步 → AI 收盤分析 → 知識沉澱 → 深度研究。每個模組的產出都餵養下一個模組。

📈

為什麼值得用

系統的價值隨使用時間累積。用越久，策略大腦的規則越準確，AI 分析的 context 越完整，投資決策品質的提升是可量化的。
